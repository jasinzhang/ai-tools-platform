# 🔧 Gemini API 网络连接问题解决方案

## ⚠️ 问题说明

如果看到错误：`Network error: Unable to connect to Gemini API`

这通常意味着：
1. 无法访问Google的API服务器
2. 可能需要代理或VPN
3. 或者API端点URL需要更新

## 🛠️ 解决方案

### 方案1: 检查网络连接

在中国大陆，可能需要：
- 使用VPN/代理访问Google服务
- 或者使用支持国际网络的网络环境

### 方案2: 使用代理配置（如果可用）

如果有代理，可以在启动服务器前设置：

```bash
export HTTP_PROXY=http://your-proxy:port
export HTTPS_PROXY=http://your-proxy:port
npm start
```

### 方案3: 测试本地工具（不需要网络）

先测试不需要AI的工具：
- 密码生成器: http://localhost:3000/tools/password-generator.html
- 二维码生成器: http://localhost:3000/tools/qr-generator.html
- 配色方案生成器: http://localhost:3000/tools/color-palette.html

这些工具不依赖外部API，应该可以正常工作。

### 方案4: 使用其他AI提供商

如果Gemini API无法访问，可以考虑：
- 使用OpenAI API（如果可访问）
- 或者配置其他AI服务

## 🔍 诊断步骤

1. **检查网络连接**:
   ```bash
   ping google.com
   ```

2. **测试API端点**:
   ```bash
   curl -I https://generativelanguage.googleapis.com
   ```

3. **检查防火墙/代理设置**

## 📝 临时解决方案

如果API暂时无法访问，网站的基本功能仍然可以工作：
- ✅ 首页可以显示
- ✅ 工具页面可以打开
- ✅ 非AI工具可以正常使用
- ❌ AI生成工具会显示错误（但不会影响页面加载）

等网络问题解决后，AI功能会自动恢复。

---

**建议：先测试不需要网络的功能，确认服务器正常，然后再处理API连接问题。**
