# 🚀 海外服务器部署指南

## ✅ 为什么海外服务器不需要代理？

部署在海外服务器（如AWS、DigitalOcean、Vercel等）后：
- ✅ 服务器可以直接访问Google API
- ✅ 不需要配置代理
- ✅ 访问速度更快
- ✅ 更稳定可靠

## 📋 部署前准备

### 1. 环境变量配置

在部署平台上设置以下环境变量：

```env
PORT=3000
NODE_ENV=production
AI_PROVIDER=google
GOOGLE_API_KEY=你的API密钥
# 不需要配置代理！
```

### 2. 代码优化

- ✅ 确保 `.env` 文件在 `.gitignore` 中（不要提交密钥）
- ✅ 检查所有依赖都已安装
- ✅ 测试所有功能正常

### 3. 域名配置（可选）

- 购买域名（如 namecheap.com）
- 在部署平台配置域名
- 自动获得SSL证书

## 🚀 快速部署到Vercel（推荐）

### 步骤1: 准备代码

确保代码已推送到GitHub：
```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin 你的GitHub仓库地址
git push -u origin main
```

### 步骤2: 部署到Vercel

1. 访问 https://vercel.com
2. 使用GitHub登录
3. 点击"New Project"
4. 选择你的GitHub仓库
5. 配置环境变量：
   - `GOOGLE_API_KEY` = 你的API密钥
   - `AI_PROVIDER` = google
   - `NODE_ENV` = production
6. 点击"Deploy"

### 步骤3: 完成！

部署完成后，你会得到一个URL，例如：
```
https://your-project.vercel.app
```

**就这么简单！** 🎉
