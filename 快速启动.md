# 🚀 快速启动指南

## 方法1: 使用启动脚本（最简单）

```bash
./start.sh
```

或者：

```bash
bash start.sh
```

## 方法2: 使用 npm

```bash
npm start
```

## 方法3: 使用 node

```bash
node server.js
```

---

## ✅ 启动成功的标志

启动成功后，你会看到：

```
🚀 Server running on http://localhost:3000
📱 Environment: development
```

## 🌐 如何在浏览器中访问

1. **打开浏览器**（Chrome、Safari、Firefox等）
2. **在地址栏输入**：
   ```
   http://localhost:3000
   ```
3. **按回车键**

## 📱 测试工具页面

启动后访问以下链接：

- **首页**: http://localhost:3000
- **密码生成器**: http://localhost:3000/tools/password-generator.html
- **TikTok标题生成器**: http://localhost:3000/tools/tiktok-title.html

## ❗ 如果无法打开页面

### 检查1: 服务器是否真的在运行？

在终端中应该看到：
```
🚀 Server running on http://localhost:3000
```

如果没有看到，说明服务器没有启动成功。

### 检查2: 是否有错误信息？

查看终端中是否有红色错误信息。

### 检查3: 端口是否被占用？

```bash
# 检查端口
lsof -i :3000
```

如果端口被占用：
```bash
# 停止占用端口的进程
kill $(lsof -t -i:3000)
```

### 检查4: 浏览器输入是否正确？

确保输入的是：`http://localhost:3000`

**不是**：
- ❌ `localhost:3000`（缺少 http://）
- ❌ `http://127.0.0.1:3000`（虽然这个也可以，但推荐用localhost）
- ❌ `https://localhost:3000`（不应该用 https）

### 检查5: 运行测试脚本

在**另一个终端**窗口运行：

```bash
node test-server.js
```

如果所有测试都是 ✅，说明服务器正常，问题可能在浏览器。

### 检查6: 浏览器问题

1. **尝试其他浏览器**（Chrome、Safari、Firefox）
2. **清除浏览器缓存**（Cmd+Shift+R 或 Ctrl+Shift+R）
3. **检查浏览器控制台**（F12 -> Console 标签）

---

## 🔍 诊断步骤

如果仍然无法打开，按以下步骤操作：

1. **停止服务器**（如果正在运行）：按 `Ctrl+C`

2. **运行测试脚本**：
   ```bash
   node test-server.js
   ```

3. **查看输出**：
   - 如果所有测试都是 ✅，说明服务器正常
   - 如果有 ❌，说明有问题

4. **告诉我结果**：
   - 测试脚本的输出
   - 浏览器显示的错误信息（如果有）
   - 服务器启动时的完整输出

## 💡 常见错误解决

### 错误: "Cannot find module"
```bash
npm install
```

### 错误: "Port 3000 already in use"
```bash
kill $(lsof -t -i:3000)
```

### 错误: "EADDRINUSE"
端口被占用，使用其他端口：
```bash
# 编辑 .env 文件，设置
PORT=3001
```
